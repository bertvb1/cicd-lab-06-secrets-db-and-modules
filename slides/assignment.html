Solo · Part 2 · 2.1
PART 2 · 2.1
Build the tst scope overlay
Two files: the scope descriptor, and one DB override that redirects test to its own database.
1
Create the scope descriptor services/config/resources/tst/config-mode.json with "parent": "core" (exact contents on the right).
2
Create the DB override: copy dev/ignition/database-connection/TimescaleDB/ to tst/…/TimescaleDB/ and change the connectURL database to ignition_tst.
3
Sanity-check locally before shipping: git status should show exactly the new tst/ folder — nothing else.
# 1 — services/config/resources/tst/config-mode.json
{
  "title": "tst",
  "description": "tst",
  "enabled": true,
  "inheritable": true,
  "parent": "core"
}

# 2 — copy dev's override, then edit connectURL
cp -r services/config/resources/dev/ignition/database-connection/TimescaleDB \
      services/config/resources/tst/ignition/database-connection/TimescaleDB
# connectURL: jdbc:…/ignition_dev → jdbc:…/ignition_tst
The bug you're fixing

A missing overlay doesn't error — it inherits. Until tst/ exists, the test gateway happily writes test data into whatever core points at. Silent fallback is the sharp edge.

No hand-edits on the gateway, no config uploads. The overlay is code — it travels like code.

Optional stretch
OPTIONAL · 1
Stretch 1 — fan out to a fleet, canary first
One release, two prod gateways — then make the rollout stop itself when the canary fails.
1
docker compose --profile fanout up -d → prod-b starts on :8092.
2
Extend release.yml's deploy job with a matrix over [prod, prod-b], re-run the release, verify both gateways report the same digest.
3
Canary: set max-parallel: 1 and order the matrix [prod-b, prod] so prod-b takes the release first.
4
Fail prod-b's healthcheck on purpose (stop the DB?) and watch the rollout halt before touching prod.
# release.yml — the deploy job grows a matrix
deploy:
  needs: promote
  strategy:
    matrix:
      gateway: [prod-b, prod]  # canary first
    max-parallel: 1            # one at a time
    fail-fast: true            # failure HALTS rollout
  runs-on: [self-hosted, lab06]
  environment: lab-gateway-${{ matrix.gateway }}
  steps:
    - run: scripts/deploy-image.sh ${{ matrix.gateway }} $TAG
The rule

A failed healthcheck must halt the rollout — never skip the box. That's what fail-fast + max-parallel: 1 buy you.
